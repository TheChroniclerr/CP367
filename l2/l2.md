1. When using ls -la , the files are listed with a number of different items on the line. Identify what the following means:
sample data
```css
/home/baox7248/CP367> ls -la
total 16
drwxr-xr-x.  5 baox7248 ad_user   79 Jan 15 16:33 .
drwx------. 10 baox7248 ad_user 4096 Jan 15 16:33 ..
drwxr-xr-x.  2 baox7248 ad_user   30 Jan 15 16:33 .devcontainer
drwxr-xr-x.  8 baox7248 ad_user 4096 Jan 15 16:33 .git
-rw-r--r--.  1 baox7248 ad_user    7 Jan 15 16:33 .gitignore
drwxr-xr-x.  2 baox7248 ad_user   48 Jan 16 12:21 l1
-rw-r--r--.  1 baox7248 ad_user   55 Jan 15 16:33 README.md
```
- the first character on the line.
File type. E.g. `d` → directory, `-` → regular file.
- in general terms, the next nine characters on the line.
Permission.
```css
[ owner ][ group ][ others ]
   rwx      r-x        r-x
```
`r` → read
`w` → write
`x` → execute (or “enter” for directories)
`-` → permission not granted
- the numbers in the sixth column (after the student/users column).\
File size in bytes.
- what do the . and .. entries mean?
`.` → current directory
`..` → parent directory
2. Determine what the following commands do, and their basic syntax:
- cd
```bash
cd [path]
```
Change directry. Absolute or relative.
- rm
```bash
rm [options] [filepath]
```
Remove. Remove file.
- mkdir
```bash
mkdir [options] [dirpath]
```
Make directory. Make folder with name in path.
- rmdir
```bash
rmdir [dirpath]
```
Remove (empty) directory. Fails if the directory is not empty.
- head
```bash
head [options] [filepath]
```
Display first 10 lines of a file.
- tail
```bash
tail [options] [filepath]
```
Display last 10 lines of a file.
